The problem

Our client for this project is an alumni association that had the following problem : They have a platform called AlumForce which is a professional social network for former students of the school, listing their profiles to connect them, store their data, allow them to view other users' profiles, or access job offers. The problem is that AlumForce is a proprietary platform owned by another company and the Alumni association does not have control over it, and has no control over its functional evolution. They cannot choose the next developments of the application and are stuck if they want to add other types of data than those provided by the developers of AlumForce. This creates a strong dependence on the solution's publisher and limits the system's ability to adapt to the association's particular uses. The Alumni therefore wish, alongside their use of AlumForce, to have a web platform that they own, on which they have control, in order to customize it as much as possible and add data that cannot be entered on AlumForce. This is why CGI was commissioned to create this application.

Implemented features

The following features were implemented :
  • The ability to regularly retrieve and update data from AlumForce using their API.
  • The ability to add and modify data on each profile.
  • The ability to perform multi-criteria searches on the list of alumni.
  • The ability to efficiently and easily view alumni profiles.
  • The ability to manage access rights to limit usage to certain staff members.

Constraints

Some constraints were identified :
  • Existing data must not be duplicated or re-imported during each synchronization with AlumForce.
  • Compliance with confidentiality and personal data protection standards (GDPR*).
  • The application must be modular to easily accommodate new features in the future.

Technological choices

The choice of technologies focused on an architecture with Angular for the front-end and Spring Boot for the back-end, along with a PostgreSQL relational database. For the authentication system, we used JWT tokens. These technical choices meet the constraints of security, robustness, maintainability, and compatibility with CGI's internal environments.

Critique and conclusion

This project was a great opportunity to work on a full-stack web application using modern technologies. It was successfully deployed and is currently in use by the alumni association. However, there are areas for improvement, for example, we could have connected the application to the real LinkedIn API to fetch and update user data directly from LinkedIn profiles. This would have eased the data entry process for users. Overall, this project allowed me to enhance my skills in web development and work collaboratively in a team setting.
